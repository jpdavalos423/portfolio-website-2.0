## Overview

Transfer Pathway Planner is a full-stack web application designed to help students map an efficient transfer path from California community colleges to UC Computer Science programs. The experience focuses on turning degree and prerequisite complexity into a clear, actionable plan.

## Problem

Transfer planning is often fragmented across multiple curriculum pages, articulation agreements, and manual advising workflows. Students can miss prerequisite dependencies or take courses in the wrong order, which delays transfer readiness and increases uncertainty.

## What I Built

I built a student-facing planner with a React and TypeScript front end backed by FastAPI services and SQL-driven program data. Users can explore pathway recommendations, validate course selections, and iteratively adjust plans while receiving immediate feedback from backend validation rules.

## Technical Decisions

I structured the backend around prerequisite-aware scheduling and validation workflows so the planner could enforce course ordering constraints while remaining responsive. On the frontend, I prioritized a clean, mobile-friendly UI and fast interactions to keep planning sessions intuitive for students with different levels of advising support.

## Impact

The project demonstrates how practical software tooling can reduce friction in academic planning by transforming complex transfer requirements into guided decisions. It also strengthened my experience building end-to-end systems that combine algorithmic logic, data modeling, and user-centered interface design.
