## Overview

Built a shared observability interface for engineering teams to triage service regressions without jumping across multiple vendor tools.

## Problem

Each team maintained separate dashboards and alert routing, which increased incident triage time and created inconsistent escalation behavior.

## What I Built

An internal web app that normalized telemetry sources, surfaced alert context, and linked incidents to deployment timelines.

## Technical Decisions

I prioritized server-rendered pages and query caching for fast operator workflows under production load.

## Impact

The platform became the default starting point for production incident investigations.
