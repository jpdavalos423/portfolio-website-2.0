## Overview

Created a focused evaluation workspace for testing retrieval changes before shipping them into production question-answering flows.

## Problem

Retrieval improvements were hard to validate consistently because experiments were scattered across notebooks and ad hoc scripts.

## What I Built

A small web UI and API for managing datasets, prompts, metrics, and scored experiment runs.

## Technical Decisions

I chose a file-light local-first architecture so teams could run evaluations without standing up additional infrastructure.

## Impact

The workbench made retrieval regressions visible earlier in the development cycle.
